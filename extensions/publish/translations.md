Canonical source: [https://dokki.one/pub/docs/translations](https://dokki.one/pub/docs/translations)

Published sites can offer documents in more than one language. Visitors choose a language from the public site, while the original language remains available.

## Configure languages

- Open **Publish → Site settings**.
- Choose the original language of the site.
- Add the additional languages visitors should be able to select.
- Save the settings and publish or update the relevant documents.

## What visitors see

Visitors can switch languages while browsing the site. Dokki keeps the selected language when they follow links within the published site. If a translation is not ready or the requested language is unavailable, the original content is shown.

## When translations update

After you publish a document or add a language, translation may take a short time. Publish the latest document version first, then check the translated public page. A later source update requires a new published version before its translation can update.

## What is translated

- Document titles and body content.
- Supported home-page and navigation text.
- Page titles, excerpts, breadcrumbs, and navigation labels when available.

Structured content inside tables and Artifacts may not be translated in the same way as document text. Review those resources separately before offering a translated site.

## Search and language links

Published pages identify their available language versions so search engines and visitors can reach the appropriate page. Keep the original language and translated pages aligned by publishing important source updates promptly.
