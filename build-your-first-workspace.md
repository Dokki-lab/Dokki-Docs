## Visual guide

[Home Workspace Map](home-workspace-map.md)

## 1. Create or choose a workspace

Use the workspace switcher to create a personal workspace, or create an organization workspace when its policy allows your organization role. Choose **Blank**, a system template, a public custom template, or one of your own templates.

A blank workspace is empty. A nonblank workspace is returned immediately with a seeding status, then folders, documents, tables, artifacts, and supported agent configuration arrive in the background. Keep the workspace open while a large template finishes seeding.

## 2. Start from Home

Home is the working dashboard for the active workspace. Use its floating composer to start a Copilot or agent conversation with workspace context. The dashboard then surfaces recent resources, pins, agent-attention reminders, activity, and connections.

On an empty workspace, Editors and Admins see a dismissible setup panel. It can create a starter document, table, or artifact, and links to AI, app connections, and member invitation. Its dismissed and completed-step state is stored locally for that person and workspace.

## 3. Create or add a resource

A resource can be a document, table, artifact, file, folder, or agent. Use **New** in the sidebar, upload a file, move resources into folders, or ask Copilot to create one.

## 4. Collaborate deliberately

Invite workspace members for the normal baseline. For one resource, use direct sharing, private access, or a link with the intended view, comment, or edit level. A private resource remains hidden from broad workspace browsing until its owner, a direct role, or an approved access request grants entry.

## 5. Bring in AI and apps

Copilot can search workspace knowledge, edit documents, create tables and artifacts, and use connected apps. Agents can be installed from blueprints, configured with memory and skills, and run from chat, schedules, automations, or MCP workflows.

## 6. Publish when ready

Publishing freezes a document, table, or artifact into the public site. Later source edits stay private until you publish again or commit the current changes. Configure navigation, languages, SEO, and domains from **Publishing**.
