Install an agent from a blueprint, then adjust its instructions and access before using it for recurring work.

## Install an agent

- Go to **Workspace → Agents → Blueprints**.
- Open a blueprint and review the role, instructions, skills, and suggested connections.
- Choose personal or organization use, name the agent, and select a home workspace when requested.
- Complete installation and open the agent's control panel.

## Write useful standing instructions

Describe the agent's purpose, expected outputs, constraints, and when it should ask for approval. Keep credentials and private keys out of instructions and memory.

## Limit workspace access

Open **Workspaces** and allow only the workspaces the agent needs. A home workspace stores setup material; it does not automatically grant access to every resource.

## Add skills and pinned context

Use **Skills** for repeatable procedures. Use **Pinned** for a small set of resources the agent should consider often. Update or remove them when the process changes.

## Connect external tools

Use the **MCP** tab when the agent needs an external service. Follow the connection's sign-in instructions and grant only the access required for the task.

## Pause or uninstall

Pause an agent when it should stop accepting new work. Review schedules separately. Before uninstalling, save any outputs or instructions you need to keep, because uninstalling removes the installed agent.
