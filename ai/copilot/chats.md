# Chats

Canonical source: [https://dokki.one/pub/docs/chats](https://dokki.one/pub/docs/chats)

Chats is the home for your Copilot and agent conversations. Use it to start work, return to earlier threads, and follow requests that are still running.

## Start a conversation

- Open **Chats** or the AI panel.
- Choose Dokki Agent or another installed agent.
- Select the workspace context when the conversation is not already attached to one.
- Write the request and add any relevant resources or files.

## Use the conversation list

The list separates loading, empty, and error states so you can tell whether there are no conversations or the list has not loaded yet. Use filters and timestamps to find the thread you need.

## Saved and temporary conversations

Saved conversations remain available in the list. A temporary conversation is useful for work you do not need to keep; do not use it when the thread will become part of a longer project handoff.

## Realtime voice calls

In a supported conversation, use the phone control to start a Realtime voice call. The call view shows connection status, duration, transcript context, mute, and hang-up controls.

- Allow microphone access when your browser asks.
- Use mute when you want to keep the call open without sending audio.
- Hang up before leaving the conversation.
- Use the transcript to review what was discussed after the call.

## When a conversation is still working

Tool use and longer agent tasks may continue after the first response appears. Follow the status in the conversation. Avoid sending a second request that changes the same resource until the first one has finished or stopped.

## Troubleshooting

- If the list cannot load, retry before creating a duplicate conversation.
- If voice does not connect, check microphone permission and try again from the same conversation.
- If a resource is unavailable, confirm that the selected workspace and your access are still correct.
