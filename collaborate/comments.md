## Create a thread

1. Select text in the document.
1. Use the comment control in the floating toolbar, or start a thread from the comments panel.
1. Write the note and post it.

The selected text receives a highlight. Selecting that highlight focuses the matching thread in the comments panel.

## Reply, resolve, and reopen

Threads are either **open** or **resolved**. Open threads accept replies. Resolve a thread when the discussion is finished; use the status control to reopen it when more work is needed. Resolved threads remain available when the panel includes resolved comments.

You can edit your own original comment. You can delete your own thread or reply; a resource admin, the resource creator, or a workspace admin can delete other people's comments. Deleting a top-level thread also deletes its replies and removes its text highlight.

## Mentions and tags

Use @mentions in a new comment or reply to notify people. Dokki sends each mentioned person a notification that links back to the thread.

Comment threads reuse the workspace's tag vocabulary. Anyone who can comment can attach or remove an existing workspace tag from a thread. Creating, renaming, or deleting workspace tags requires an Editor or Admin workspace role. Tags are useful for review states such as legal, design, blocked, or follow-up.

## Permissions

Comment access can come from a direct **Commenter**, **Editor**, or **Admin** role; creator status; a link with comment or edit access; or workspace membership on a normal resource. Private resources still require direct access or admin authority.

The comments panel records filters per resource. Use its filter controls to focus the discussion you need, and reset filters to return to the default view.
