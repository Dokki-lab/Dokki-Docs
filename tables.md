Dokki tables are typed, collaborative resources for structured work. They use one live Y.js table document, so edits in every view affect the same rows and columns.

- **Column Types** - text, number, boolean, date, date range, select, multi-select, tags, member, URL, and email.
- **Filters & Sort** - type-aware filters, AND/OR groups, one active sort, hidden and frozen columns, and text wrapping.

- **Forms** - public collection forms that append validated rows.

## What tables are for

Use tables for trackers, content calendars, CRM-style lists, research logs, intake queues, lightweight roadmaps, and operational work that needs both structure and narrative context. Copilot can create or transform rows, while people edit in the same live table.

## Views and personal presentation

The view switcher offers Table, Kanban, Gantt, and Calendar. Filters, sort, frozen or hidden columns, widths, wrapping, and view-specific configuration affect how the current browser displays the table; they do not create another copy of its data.

Saved views are named personal interface presets stored in the browser's table-view state. They are useful for returning to a workflow on the same browser, but are not shared table configuration and are not a way to publish a team-wide canonical view.

## Inline versus standalone tables

The document slash command named **Table** creates an inline rich-text table inside that document. A standalone collaborative table is a separate workspace resource, created from the workspace resource flow or by Copilot, and can then be referenced from a document with **Embed Resource**.

## Current limits

Dokki tables do not yet provide formulas across cells, relational links between tables, or a dedicated file/attachment column type. Use files, documents, or separate tables where those workflows fit better.
